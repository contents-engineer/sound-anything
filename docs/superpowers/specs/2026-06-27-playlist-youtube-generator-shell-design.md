# 플레이리스트 유튜브 제너레이터 — 멀티 도구 셸 설계

작성일: 2026-06-27

## 배경 / 목적

현재 앱은 `app/page.tsx` 한 페이지에 Suno 가사 생성기 전체가 들어있는 단일 도구 앱이다.
이를 **"플레이리스트 유튜브 제너레이터"** 라는 상위 타이틀(앱 셸) 아래 여러 도구를 메뉴로
전환하는 멀티 도구 구조로 바꾼다.

- 메뉴 1: **수노 제너레이터** — 현재 기능 그대로
- 메뉴 2: **제미나이 워터마크 제거** — 슬롯(플레이스홀더)만. 기능 없음.

## 접근 방식

**Next.js App Router 라우트로 분리** (클라이언트 탭 상태 방식이 아님).

- 메뉴 = URL → 새로고침·북마크·공유가 자연스럽고, 각 도구가 독립 파일로 격리되어 유지보수가 쉽다.
- 사이드바는 루트 레이아웃에서 공유되며 `usePathname`으로 활성 메뉴를 강조한다.
- Next.js 16.2.6 App Router 규칙(파일 기반 라우팅, 루트 레이아웃 html/body 필수, `next/link`,
  중첩 레이아웃)에 따른다.

대안으로 검토한 "단일 페이지 + 클라이언트 탭 상태"는 URL이 없어 공유/북마크가 불가하고
도구가 늘수록 한 파일이 비대해져 제외했다.

## 파일 구조

```
app/
  layout.tsx        # 루트 레이아웃: html/body + 사이드바 셸 + {children}. metadata 타이틀 변경
  page.tsx          # 수노 제너레이터 ("/"). 기존 로직 그대로, 헤더만 소폭 조정
  watermark/
    page.tsx        # 제미나이 워터마크 제거 ("/watermark"). 플레이스홀더, 서버 컴포넌트
components/
  Sidebar.tsx       # (신규) 클라이언트 컴포넌트. 앱 타이틀 + 메뉴 2개 + 활성 강조
```

기존 컴포넌트(`OptionSection`, `ResultPanel`, `HistoryDrawer`, 등)와 `lib/`, `app/api/`는
변경하지 않는다.

## 컴포넌트 설계

### `components/Sidebar.tsx` (신규, 클라이언트)
- 역할: 앱 대제목 + 메뉴 네비게이션 렌더링, 현재 경로에 따라 활성 항목 강조.
- 의존: `next/link`의 `Link`, `next/navigation`의 `usePathname`.
- 메뉴 데이터(이 컴포넌트 내 상수):
  | label | href |
  |---|---|
  | 수노 제너레이터 | `/` |
  | 제미나이 워터마크 제거 | `/watermark` |
- 활성 판정: `usePathname() === item.href` (정확 일치). 루트 `/`는 정확 일치로 충분.
- 활성 항목은 기존 테마색(violet, 예: `bg-violet-600 text-white` 또는 violet 텍스트/좌측 보더)으로 강조.
- 상단: `🎵 플레이리스트 유튜브 제너레이터` 타이틀.

### `app/layout.tsx` (수정)
- `html`/`body`는 유지(루트 레이아웃 필수 요건).
- `body` 안에 가로 플렉스 셸: `<Sidebar />` + `<main>{children}</main>`.
- 데스크톱(md+): 사이드바 좌측 고정 폭(~240px) + sticky, 콘텐츠는 우측에서 스크롤.
- 모바일(md 미만): 사이드바가 상단 가로 바로 전환(콘텐츠를 가리지 않음).
- `metadata.title`을 `플레이리스트 유튜브 제너레이터`로 변경. description도 멀티 도구에 맞게 조정.

### `app/page.tsx` (수정 — 수노)
- 생성/재생성/히스토리 등 **로직은 변경 없음**.
- 현재 앱 전체 타이틀 역할을 하던 `<h1>Suno.ai Lyrics Generator</h1>`를
  섹션 제목 `수노 제너레이터`로 변경(대제목은 사이드바로 이동).
- 히스토리 버튼과 부제는 그대로 유지.
- 컨테이너(`mx-auto max-w-5xl px-4 py-8`)는 콘텐츠 영역 안에서 자연스럽게 동작하도록 조정.

### `app/watermark/page.tsx` (신규 — 플레이스홀더)
- 서버 컴포넌트. 자체 `metadata`(title: `제미나이 워터마크 제거`) export.
- 가운데 정렬 카드: 제목 `제미나이 워터마크 제거` + "준비 중입니다" 안내 문구.
- 기능/입력/API 호출 **없음**.

## 스타일

기존 zinc/violet 팔레트와 `rounded-xl`/`rounded-2xl` 톤을 그대로 재사용해 일관성 유지.
새 색·폰트 도입 없음.

## 동작 흐름

1. 사용자가 어떤 경로로 진입 → 루트 레이아웃이 사이드바 + 해당 페이지를 렌더.
2. 사이드바 메뉴 클릭 → `<Link>` 클라이언트 전환, `usePathname`으로 활성 강조 갱신.
3. `/` → 기존 수노 생성기 전체 동작(변경 없음).
4. `/watermark` → "준비 중" 플레이스홀더 표시.

## 비범위 (Out of scope)

- 워터마크 제거 실제 기능(이미지 업로드, Gemini 호출, 처리 로직) — 슬롯만.
- 인증, 라우트별 권한, 다국어 라우팅.
- 기존 수노 생성 로직 변경.

## 성공 기준

- `/`에서 기존 수노 생성기가 이전과 동일하게 동작한다(회귀 없음).
- 좌측 사이드바에 앱 대제목과 두 메뉴가 보이고, 현재 메뉴가 강조된다.
- 메뉴 클릭으로 `/`와 `/watermark`를 오갈 수 있다.
- `/watermark`는 "준비 중" 플레이스홀더를 보여준다.
- `npm run build` / `lint` 통과.
- 모바일 폭에서 사이드바가 콘텐츠를 가리지 않는다.
